# Kano Model

A feature classification framework that categorizes product attributes based on the nonlinear relationship between feature implementation and customer satisfaction. The model reveals that not all features are equal — some delight, some are expected, and some only matter when absent. This helps teams allocate resources to the features that create disproportionate value.

## Steps

1. **List Candidate Features** — Gather the full set of features, improvements, or attributes under consideration. Include both team-generated ideas and customer-requested items.

2. **Design the Kano Questionnaire** — For each feature, create a functional question ("How would you feel if this feature were present?") and a dysfunctional question ("How would you feel if this feature were absent?"). Use a 5-point scale: Like, Expect, Neutral, Tolerate, Dislike.

3. **Collect Responses** — Survey target customers or stakeholders. Ensure a representative sample across key segments. Record the functional/dysfunctional answer pair for each feature per respondent.

4. **Classify Each Feature** — Cross-reference functional and dysfunctional answers using the Kano evaluation table to categorize each feature as: Must-Be (basic), One-Dimensional (performance), Attractive (delight), Indifferent, Reverse, or Questionable.

5. **Calculate Category Strength** — For each feature, compute the percentage of respondents in each category. Determine the dominant category and calculate the Customer Satisfaction (CS) coefficient: CS+ = (Attractive + One-Dimensional) / Total, CS- = (Must-Be + One-Dimensional) / Total.

6. **Prioritize the Roadmap** — Must-Be features are non-negotiable table stakes (fix first). One-Dimensional features compete on execution (invest proportionally). Attractive features are differentiation opportunities (strategic bets). Indifferent features are deprioritized.

## Output Format

A structured report containing:

- **Feature List**: All evaluated features with their dominant Kano category
- **Classification Matrix**: Distribution of respondent answers per feature across all Kano categories
- **CS Coefficient Chart**: Each feature plotted by CS+ (satisfaction increase) and CS- (dissatisfaction if absent)
- **Priority Tiers**: Features grouped into Must-Build, Performance Drivers, Delighters, and Deprioritized
- **Strategic Recommendations**: Which features to invest in, maintain, or skip — with rationale tied to category and coefficient scores
