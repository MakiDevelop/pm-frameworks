# Hook Model

A behavioral design framework for building products that form user habits through a four-phase loop: Trigger, Action, Variable Reward, and Investment. Created by Nir Eyal, the model explains how products like social media, messaging apps, and marketplaces create internal triggers that bring users back repeatedly — reducing dependence on advertising and increasing lifetime value.

## Steps

1. **Identify the Internal Trigger** — Determine the emotional state or routine that should eventually cue product usage without any external prompt. Common internal triggers: boredom, anxiety, loneliness, uncertainty, FOMO. Map the target user's pain point or desire that the product relieves.

2. **Design the External Trigger** — Define the external cues that initially prompt the user to act before the habit forms. These include paid triggers (ads), earned triggers (press, SEO), relationship triggers (word of mouth), and owned triggers (notifications, emails). Plan how to transition users from external to internal triggers over time.

3. **Simplify the Action** — Make the intended behavior as easy as possible. Apply Fogg's Behavior Model: Behavior = Motivation x Ability x Trigger. Reduce friction by minimizing steps, cognitive load, and time. The action should be simpler than thinking about whether to do it.

4. **Deliver Variable Rewards** — Provide a reward that satisfies the user's need but includes an element of variability to maintain interest. Three types: Rewards of the Tribe (social validation — likes, comments), Rewards of the Hunt (material gain — deals, information), Rewards of the Self (personal mastery — completion, skill). The variability is what sustains engagement.

5. **Prompt an Investment** — Ask the user to put something back into the product that improves the next cycle: data, content, followers, reputation, or skill. Investments increase switching costs and store value that makes the product better with use. They also load the next trigger.

## Output Format

A structured report containing:

- **Internal Trigger Map**: The target emotional state or user pain point, with evidence from research or persona analysis
- **External Trigger Plan**: Channels and mechanisms for each trigger type (paid, earned, relationship, owned) with transition strategy toward internal triggers
- **Action Audit**: Current and proposed user flow for the core action, with friction points identified and simplification recommendations
- **Variable Reward Design**: Specific reward mechanics mapped to Tribe, Hunt, and Self categories, with variability mechanisms described
- **Investment Loop**: What the user invests, how it stores value, and how it loads the next trigger
- **Hook Cycle Diagram**: A visual summary of the complete loop showing how each phase connects to the next
- **Ethical Check**: Assessment of whether the habit serves genuine user needs or exploits vulnerabilities
